### **Project Objective**
The objective of this project is to analyze customer churn behaviour, revenue generation, payment trends, and plan performance using telecom subscription data.<br>
This project aims to :<br>
•	Measure churn rate<br>
•	Analyze payment delays<br>
•	Determine most profitable plans<br>
•	Provide business-driven recommendations<br>
•	Understand revenue distribution<br>
•	Compare active vs churned customers<br>
•	Provide actionable business insights<br>

### **Tools & Technologies Used**
•	Python (Pandas, NumPy)<br>
•	Power BI (DAX, Data Modeling,KPI)<br>
•	Jupyter Notebook<br>
### **Project architecture**
Python (Cleaning & Feature Engineering)<br>

CSV Export<br>
↓<br>

↓<br>

↓<br>

↓<br>
Business Insights<br>

### **Data Collection**
Used 4 datasets :<br>
Customers (10,000 records)<br>
Payments (30,000 records)<br>
Subscriptions (10,000 records)<br>
Plans (3 records)

### **Data Cleaning & Preprocessing (Python – Pandas)**
Performed data cleaning using Jupyter Notebook :<br>
Removed duplicate records<br>
Handled missing values<br>
Fixed inconsistent text (e.g., gender, region)<br>
Converted incorrect data types (dates, age)<br>
Standardized categorical values<br>
Replaced invalid values with mean/appropriate values

### **Feature Engineering**
Created meaningful fields like :<br>
Age groups<br>
Late payment flags<br>
Customer segmentation improvements

### **Data Modeling (Power BI)**
Built relational model :<br>
Customers → Subscriptions<br>
Customers → Payments<br>
Plans → Subscriptions<br>
Created Date Tables for time-based analysis

### **DAX Calculations**
Developed key business metrics :<br>
Churn Rate<br>
Total Revenue<br>
Failed Payments Revenue<br>
Late Payment %<br>
Active vs Churned Customers<br>
Profit Calculation

### **Dashboard Development (Power BI)**
Created 3 interactive dashboards :<br>
Executive overview<br>
Subscription & Plan Performance<br>
Payment & Risk Analysis
### **Project impact :**
This project enables telecom management to :<br>
•	Reduce churn<br>
•	Reduce revenue leakage<br>
•	Increase customer retention<br>
•	Adjust pricing strategy<br>
•	Increase profitability<br>
•	Launch region-specific campaigns<br>
•	Focus expansion in high-performing regions<br>
•	Identify which payment mode causes higher failures


### **Conclusion**
Built an end-to-end analytics pipeline<br>
Used Python for data cleaning & transformation<br>

Delivered actionable insights for telecom business improvement
